# Revel Heroku AI Ops Monitor

A real-time AI-powered operations dashboard that monitors your Heroku application logs, automatically detects errors, and uses a local AI model to diagnose what went wrong — and suggest how to fix it.

---

## What Is This?

When you run a backend application on Heroku, errors happen constantly: failed API requests, database issues, crashes, security scans from bots, and more. Normally you'd have to dig through thousands of raw log lines to find and understand each problem.

**This tool does that for you, automatically:**

1. It connects to your Heroku app's live log stream
2. It watches for any HTTP errors (4xx, 5xx), panics, and database failures
3. It sends the relevant log context to a **local AI model** (DeepSeek Coder, running via Ollama on your machine) which reads your Go backend source code and diagnoses the root cause
4. Every diagnosis — with the raw error, a technical explanation, and a suggested fix — is saved to a log file
5. A **beautiful web dashboard** (running at `localhost:3333`) shows everything in real time, with charts, filters, and a priority task checklist generated by the AI

No data leaves your machine. The AI runs 100% locally.

---

## Dashboard Preview

The dashboard provides:

- **Live error feed** — every failed request appears as a card with status code, HTTP method, endpoint, AI diagnosis, and suggested fix
- **Error rate chart** — visualises 4xx vs 5xx errors over time
- **AI Task Checklist** — a right-side panel where the AI automatically generates a prioritised list of bugs to fix
- **Anomaly detection** — if an endpoint suddenly spikes beyond its historical baseline, it is flagged automatically
- **Filters** — filter by status code (4xx / 5xx), HTTP method, confidence level, or search by endpoint

## Prerequisites

Before running this tool you need the following installed on your machine:

### 1. Node.js
Download from [nodejs.org](https://nodejs.org). Version 18 or higher recommended.

```bash
node --version   # should print v18 or higher
```

### 2. Heroku CLI
Download from [devcenter.heroku.com/articles/heroku-cli](https://devcenter.heroku.com/articles/heroku-cli) and log in:

```bash
heroku login
heroku whoami   # should print your email
```

### 3. Ollama (local AI runtime)
Download from [ollama.com](https://ollama.com) and install the DeepSeek Coder model:

```bash
ollama pull deepseek-coder:6.7b
ollama run deepseek-coder:6.7b   # test it works, then Ctrl+C to exit
```

Ollama must be running in the background whenever you use the monitor.

## Setup

### 1. Clone the repository

```bash
git clone https://github.com/YOUR_USERNAME/revel-heroku-ai-monitor.git
cd revel-heroku-ai-monitor
```

### 2. Configure the monitor

Open `monitor.js` and update these two constants near the top of the file:

```js
const APP_NAME = 'your-heroku-app-name';   // your Heroku app slug
const BACKEND_PATH = 'C:\\path\\to\\your\\go\\backend'; // absolute path to your Go source code
```

- **`APP_NAME`** — the name of your Heroku app (the part in `your-app.herokuapp.com`)
- **`BACKEND_PATH`** — the folder where your Go backend lives. The monitor reads `.go` files from a `handlers/` subdirectory inside this path to give the AI source code context. If you don't have a Go backend, set this to any empty folder and the AI will still work without source context.

### 3. (Optional) ngrok tunnel

If you want to access the dashboard from another device or share it remotely, copy `ngrok.yml.example` to `ngrok.yml` and fill in your own ngrok authtoken from [dashboard.ngrok.com](https://dashboard.ngrok.com).

> **Important:** never commit `ngrok.yml` to git — it contains a secret token. It is already in `.gitignore`.

## Running the Monitor

Make sure Ollama is running first:

```bash
ollama serve   # keep this terminal open
```

Then in a separate terminal, start the monitor:

```bash
node monitor.js
```

You will see output like:

```
🗺️  ROUTE_MAP auto-descubierto: 18 rutas
⏳ Conectando con Heroku Stream para "revel"...
🌐 Dashboard Ops Center listo en: http://127.0.0.1:3333
```

Open your browser at **http://localhost:3333** to see the dashboard.

The monitor will now:
- Stream live logs from your Heroku app
- Detect errors automatically
- Send them to DeepSeek for analysis (you will see `🤖 [Fase 1]` messages in the terminal)
- Update the dashboard in real time (it auto-refreshes every 10 seconds)

## Generated Files

The monitor creates three local files at runtime (all ignored by git):

| File | Description |
|---|---|
| `revel_ai_errors.log` | Plain-text log of every diagnosed error with timestamp, status, method, URL, AI explanation, and suggested fix |
| `revel_ai_checklist.json` | JSON array of prioritised tasks generated by the AI, displayed in the dashboard sidebar |
| `revel_ai_baseline.json` | Historical error-rate data per endpoint per hour, used for anomaly detection |

---

## How the AI Analysis Works

For each detected error the monitor runs a **two-phase AI pipeline**:

**Phase 1 — Diagnosis**
The raw log lines around the failed request are sent to DeepSeek along with relevant Go handler source code (retrieved via semantic function matching). The AI is asked to identify the exact raw error and explain why it happened.

**Phase 2 — Formatting**
The free-text diagnosis is sent back to DeepSeek with a strict JSON schema, producing a clean structured output:

```json
{
  "razonamiento": "Technical explanation in Spanish",
  "error_real": "Error 1062: Duplicate entry '...' for key 'PRIMARY'",
  "suggested_fix": "Add unique constraint check before INSERT"
}
```

This two-step approach (reflect first, then format) produces far more accurate results than asking for JSON directly.

**Anomaly detection** runs in parallel: each endpoint's error rate is tracked in 60-second windows and compared against a rolling historical average. If the current rate is more than 2× the baseline and at least 3 errors occurred, the entry is flagged as an anomaly and promoted in the checklist.

## Dashboard API Endpoints

The monitor exposes a small HTTP API used by the dashboard:

| Endpoint | Description |
|---|---|
| `GET /` | Serves the dashboard HTML |
| `GET /api/logs` | Returns the raw error log file |
| `GET /api/checklist` | Returns the AI-generated task list as JSON |
| `GET /api/baseline` | Returns the current anomaly baseline data |
| `GET /api/queue` | Returns the current AI analysis queue status |

## Adapting to Your Own Project

This tool was built for a Go backend deployed on Heroku, but it can be adapted:

- **Different Heroku app** — just change `APP_NAME`
- **Non-Go backend** — set `BACKEND_PATH` to an empty folder; the AI will still analyse logs, just without source code context
- **Different AI model** — change `MODEL_NAME` to any model available in Ollama (e.g. `codellama`, `llama3`, `mistral`)
- **Different port** — the dashboard port (`3333`) is set in the last line of `monitor.js`

## Troubleshooting

**"Heroku stream cerrado"** — The Heroku CLI is not logged in or the app name is wrong. Run `heroku login` and verify `APP_NAME`.

**"Ollama no disponible"** — Ollama is not running. Start it with `ollama serve` in a separate terminal.

**Dashboard shows no errors** — Either your app has no current errors (good!) or the `TRIGGER_REGEX` in `monitor.js` is not matching your log format. Check the terminal output for incoming log lines.

**AI output marked LOW confidence** — The model returned a vague response. This is normal for errors with very little log context. The entry is still saved.
